Our semantics works incorrectly with example below according to
[Batty-el:POPL11] formalization:

int main() {
  atomic_int x = 0;
  atomic_int z = 0;
  {{{
      { x.store(5, memory_order_release);
        z.store(1, memory_order_relaxed); }
  ||| { r1 = z.load(memory_order_acquire);
        r2 = x.load(memory_order_relaxed); }
  }}};
  return 0; }

In [Batty-el:POPL11] cppmem `r1 = 1; r2 = 0` result is possible.
